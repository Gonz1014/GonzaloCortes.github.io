Java Blackjack & Video Poker

  • The original was a super-simple Intro to Java final project console poker variant (no true Jacks-or-Better rules, clunky text I/O), so gameplay felt limited and the code wasn’t structured for reuse.
  • I wanted to push project into something I’d actually play and showcase, apply real OOP and ship a desktop UI.
  • I refactored into reusable engines (Card, Deck, HandEvaluator, PayTable), upgraded to true Jacks-or-Better with correct payouts, then added Blackjack via a compact turn/state machine and dealer logic.
  • I disliked the terminal experience, so I kept the core UI-agnostic and taught myself JavaFX to build a clean GUI (tokens/bets, hit/stand/double, draw/hold, action log); that process cemented MVC separation, deterministic state flow, and robust edge-case handling (ace 1/11, naturals, reshuffle), turning it into a polished, playable app.
